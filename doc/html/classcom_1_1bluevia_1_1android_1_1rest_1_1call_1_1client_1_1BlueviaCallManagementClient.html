Operation "makeCallSession": This operation allows to request to set up a call session between two addresses, the first participant and the second participant.

Note: Notice the network is the entity which establishes the call session. The developer may specify the incoming call identifications shown on the participant phones (an application name, the other participant identification, etc).

Parameters:
firstCallParticipantThe address of the first participant involved in the call session.
secondCallParticipantThe address of the second participant involved in the call session.

Allowed UserId instances for the two parameters above are those such as:

  • UserId type is UserId.PHONE_NUMBER and has a phone number -following the E164 format- as 'userIdValue'.

Any other UserId instance type will cause this function throws an BlueviaClientException.

Parameters:
firstParticipantDisplayedAddressThe incoming call identification that will be displayed on the first participant's phone. Optional. Type NULL for none. A default value will be specified by the network.
secondParticipantDisplayedAddressThe incoming call identification that will be displayed on the second participant's phone. Optional. Type NULL for none. A default value will be specified by the network.

Allowed UserId instances for the two parameters above are those such as:

  • UserId type is UserId.PHONE_NUMBER and has a phone number -following the E164 format- as 'userIdValue'.
  • UserId type is UserId.ALIAS and any alphanumeric string as 'userIdValue'.
  • UserId type is UserId.ALIAS and the string "anonymous" as 'userIdValue' for unknown identification.

Any other UserId instance type will cause this function throws an BlueviaClientException.

Parameters:
mediaInfoListThis parameter identifies one or more media to be applied to the participants in the call sessions. Optional. Leave empty for none. The media type will be selected by the network.
changeMediaNotAllowedThis flag specifies if the switching process across different media is not allowed.
Returns:
Identifier assigned by the server for the call session.
Exceptions:
Anexception informing about errors encountered in the request/response.
